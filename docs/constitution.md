# Проект управления умными домами

## Архитектура системы

### Облачная часть
- **MQTT Broker**: mqtt-broker, запущенный в облаке
- **Python Daemon**: Сервис для обработки данных
  - Чтение значений с mqtt-broker
  - Сохранение данных в PostgreSQL с плагином TimescaleDB
  - Поддержка актуального среза значений объектов в реальном времени
  - Предоставление API для интеграции с внешними системами (MCP-сервер, визуальные интерфейсы)

### Локальная часть
- **Контроллер**: LogicMachine
- **Язык программирования**: Lua
- **Библиотека**: Контроллер имеет собственную библиотеку управления объектами KNX, описание как создать сервис https://kb.logicmachine.net/misc/apps/, пример рабочего сервиса docs/samples/daemon.lua. Контроллер все объекты из разных протоколов приводит автоматичиски к стандарту KNX, пример описания библиотеки Lua для работы с объектами https://kb.logicmachine.net/libraries/lua/, пример файла выгрузки объектов docs/objects.json через функцию  grp.all() и конвертацию в json формат. Запись значения в объект, означает что контреллер управляет реальными физичиескими устройствами

### Потоки данных

#### От контроллера в облако:
- Состояние всех устройств
- Данные объектов KNX

#### От облака к контроллеру:
- Команды управления в формате: адрес объекта + значение

### База данных
- **СУБД**: PostgreSQL
- **Плагин**: TimescaleDB (для временных рядов)

### Интеграции
- MCP-сервер
- Визуальные интерфейсы
- Другие системы через API
